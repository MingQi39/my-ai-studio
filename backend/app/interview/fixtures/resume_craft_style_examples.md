**项目背景：** 面向多类型工业资料分散、检索不准的问题，建设私有化知识问答与证据追溯能力。  
**工作内容：**

2. **原理与机制**

3. **取舍与方案**
   - 构建向量检索与关键词检索融合、再经重排的混合召回链路，并叠加知识图谱增强召回。
4. **证据与验证**
   - 支持答案引用原文、表格、图片与图谱事实；证据不足时拒答。
5. **场景与定位**
   - 设计租户 / 部门 / 角色 / 敏感级权限过滤，适配企业私有化场景。

## Example B — 价值样

### 示例·医疗智能问诊（LLM + 知识图谱）

**角色待补充** · **时间待补充**  
**技术栈：** ReAct Agent、图查询工具、向量长期记忆、会话短时记忆、流程状态机、多模态报告理解  
**项目背景：** 在线问诊存在主诉模糊、多轮上下文易断、复诊与既往史脱节等问题，目标覆盖问诊采集、鉴别与报告解读闭环。  
**工作内容：**
1. **取舍与方案**
   - 以 ReAct 强制先查图谱再追问，抑制松散逻辑与幻觉；用轻量状态机约束问诊阶段切换。
2. **原理与机制**
   - 双层记忆：长期画像语义召回 + 短时滑动窗口，减少重复采集与无效上下文。
3. **证据与验证**
   - 口语主诉做实体链接到标准术语；报告图文结合历史记录生成结构化建议。
